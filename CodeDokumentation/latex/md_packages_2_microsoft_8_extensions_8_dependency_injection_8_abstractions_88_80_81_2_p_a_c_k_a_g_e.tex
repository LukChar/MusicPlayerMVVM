Supports the lower-\/level abstractions for the dependency injection (DI) software design pattern which is a technique for achieving Inversion of Control (IoC) between classes and their dependencies.\hypertarget{md_packages_2_microsoft_8_extensions_8_dependency_injection_8_abstractions_88_80_81_2_p_a_c_k_a_g_e_autotoc_md13}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{md_packages_2_microsoft_8_extensions_8_dependency_injection_8_abstractions_88_80_81_2_p_a_c_k_a_g_e_autotoc_md13}

\begin{DoxyItemize}
\item Interfaces for DI implementations which are provided in other packages including {\ttfamily Microsoft.\+Extensions.\+Dependency\+Injection}.
\item An implementation of a service collection, which is used to add services to and later retrieve them either directly or through constructor injection.
\item Interfaces, attributes and extensions methods to support various DI concepts including specifying a service\textquotesingle{}s lifetime and supporting keyed services.
